# F006_BATCH_REDACTION

The CA/tax beachhead workflow is many documents per client, but redaction was one file at a time. Add a `redact-batch <folder>` CLI command that redacts every supported document in a folder (PDFs plus the images/txt/md/docx that `parsers/converters.py` already ingests), writing `<name>.redacted.pdf` alongside the originals or into `--out <dir>`, and never mutating the source. Factor the detect→redact pipeline out of `cli/redact_cmd.py` into `engine/redact.py` (`detect_document`, `redact_document`) so the single-file command and the batch runner share exactly one code path (CLI stays a thin wrapper, per engine rules). Convert non-PDF inputs into a temp dir so no conversion twins litter the workspace, and exclude our own outputs (`*.redacted.pdf`, `*.certificate.*`) from the input scan. Continue-on-error: one unreadable file is recorded on its summary row, not fatal. Write a `redaction-summary.{csv,json}` with per-file status, redaction counts by type, `--certificate` verification result (F007), output path, and error. Carry the same OCR/type/values-only flags as `redact`, plus `--certificate`. Exit non-zero if any file errored or failed verification, so scripts and cron jobs catch failures. Acceptance: a folder of synthetic statements produces one redacted copy each plus a summary; a deliberately corrupt file is reported but the good files still process; a second run does not treat the first run's outputs as new inputs.
