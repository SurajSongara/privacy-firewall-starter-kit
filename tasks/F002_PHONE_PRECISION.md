# F002_PHONE_PRECISION

Close the last known false-positive class from the P004 baseline: PHONE sits at 75% precision because 10-digit UTR/Ref-ID numbers in bank statements land in the ask band. Extend the P002 context lexicons with transaction-reference vocabulary (UTR, RRN, Ref No, Cheque No, Txn ID and OCR-mangled variants) and demote 10-digit candidates hard when (a) the line context matches a reference lexicon, (b) the digits fail Indian mobile prefix rules (must start 6-9), or (c) the number is embedded in a longer digit/alphanumeric run. Promote when a phone label or +91/0 prefix is present. Acceptance: PHONE precision ≥ 0.9 on the P004 benchmark with zero recall loss on true phone numbers; the two documented UTR traps on TestFiles/statement1-5.pdf are demoted out of the ask band; benchmark baseline updated.
