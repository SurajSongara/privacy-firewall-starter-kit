# F004_WORKSPACE_MEMORY

Un-defer P010 at workspace scope — its deferral condition ("until the review UI has real usage") is met. Manual marks are currently per-document: reviewing eight resumes means marking the same name eight times. Add a per-workspace terms file (`.privacy-firewall/terms.json` in the studio workspace: `[{text, label, case_sensitive, added_from}]`, plus an allowlist for keep-decisions) maintained by the engine, never the UI. When a review session opens a document in a workspace, each term is applied like `mark_text` but with `suggested_action` from the policy and no pre-set decision — remembered terms are *suggestions* the reviewer confirms, preserving the review contract. The review UI gets a "remember for this workspace" toggle on the mark popup and search card; the studio dashboard shows term count per workspace. Opt-in, local-only, deletable. Acceptance: mark "Suraj" with remember enabled in one resume, open a second resume in the same workspace, and its instances appear pre-suggested; keep-allowlisted terms stop being suggested; sessions outside a workspace (plain `review` command) are unaffected unless a terms file is passed explicitly.
