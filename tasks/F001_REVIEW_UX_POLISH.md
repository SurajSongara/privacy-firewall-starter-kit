# F001_REVIEW_UX_POLISH

Two small trust fixes in the review flow, shipped as one batch. (1) Honest mark feedback: `ReviewSession.mark_text` counts matches it skipped as already-marked duplicates and `POST /api/mark` returns `{"added": N, "skipped": M}`; the UI toast then distinguishes "All M matches are already marked" from "Text not found" — today both collapse into the misleading "No new matches — the text was not found (or is already marked)". (2) Overlapping redaction rects: `PDFRenderer._apply_plan` dedupes *identical* rects, but two detections covering the same text with different bboxes (e.g. the EMAIL detector plus a manual mark of the same email at half width) still both draw fill + stars, producing misaligned star-on-star output. Merge REPLACE/BLACK_BAR job rects that overlap on the same line (IoU or containment above a threshold) into their union before drawing, keeping the widest rect's replacement. Acceptance: re-searching an already-marked term reports the skipped count; a plan containing detector + manual entries for the same email renders one clean star run; existing renderer tests stay green.
