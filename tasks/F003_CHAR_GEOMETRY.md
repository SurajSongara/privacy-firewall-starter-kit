# F003_CHAR_GEOMETRY

Replace proportional character interpolation with real per-character geometry from PyMuPDF's `rawdict`. Two defects motivate this. First, `ReviewSession._char_range_bbox` and the UI drag mapping assume uniform glyph widths — exact only for monospace (converted txt/md); on proportional fonts (LaTeX resumes) a sub-word redaction box can clip a neighbouring character or leave a sliver of PII unredacted, which breaks the redaction guarantee. Second, `block.text` offsets and `get_text("words")` span order disagree when glyphs interleave (observed: FontAwesome icons in a resume header reorder ahead of words, producing a manual email mark whose saved bbox is half the email's true width). Plumb per-char bboxes through the parser (or a session-level `rawdict` sidecar keyed by page), use them in `_char_range_bbox`, and extend `GET /api/text/{page}` with per-word char x-offsets so the drag selection in the review UI maps pixel positions to exact character boundaries. Acceptance: marking a substring inside a proportional-font word yields a bbox that covers exactly those glyphs (assert against `page.search_for` of the substring clipped to the word); a manual mark of the resume header email covers its full width; the partial-word drag popup pre-fills the exact substring on a LaTeX resume, not ±1 character.
