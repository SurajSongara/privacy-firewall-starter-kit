# F008_GSTIN_DETECTOR

CAs live in GST, so GSTIN is a high-relevance Indian identifier that keeps recall ahead of Western-only tools — and it is a perfect fit for the deterministic engine (structured format plus a checksum). Add `detectors/gstin_detector.py` matching the 15-character GSTIN format (2-digit state code, embedded 10-char PAN, entity character, `Z`, check character) with alphanumeric lookarounds so a GSTIN embedded in a longer token is not matched. Validate two things: the GST state code (01–38, plus 97/99) and the official base-36 check digit (walk the first 14 characters right-to-left with an alternating 2,1 factor, sum each product's base-36 digits, derive the check character from the running total). A format match plus a valid checksum is high-confidence (0.95), with the state code and "checksum valid" recorded in `reasons`. Register it in `ALL_DETECTORS` (`detectors/__init__.py`) — the single source of truth — so it flows into `detect`, `redact`, `redact-batch`, and the verifier automatically with no per-call wiring. Acceptance: published sample GSTINs validate, a mutated check digit and an invalid state code are rejected, no match inside a longer alphanumeric run, and the P004 precision benchmark for the existing seven types is unchanged.
